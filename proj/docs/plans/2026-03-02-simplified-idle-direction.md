# 简化方向：偏挂机独立游戏

> 当前设计偏重（传奇 + 动森 + MC），作为独立游戏体量过大。本文档提出偏挂机简化方向，降低开发与运营成本。

---

## 一、当前设计 vs 简化目标

| 维度 | 当前设计 | 简化目标 |
|------|----------|----------|
| 操作 | 即时战斗、技能释放、连击、走位 | 低操作：挂机/自动战斗为主 |
| 数值 | 装备、强化、镶嵌、品质、技能树、职业 | 简单等级 + 经验 + 基础属性 |
| 沙盒 | 完全可破坏、建造、红石、自动化 | 少量可破坏（树/石头）、简单采集 |
| 内容 | 多地图、副本、BOSS、公会、社交 | 单地图或 2–3 张，无副本、无公会 |
| 社交 | 组队、交易、拍卖行 | 单机优先，多人可延后或砍掉 |

---

## 二、挂机核心玩法

### 核心循环

```
玩家挂机 → 角色自动寻怪/攻击 → 获得经验/金币/掉落 → 升级/购买 → 更强 → 挂更高级的怪
```

### 操作频率

- **低**：选地图、选挂机点、偶尔点开背包/升级
- **无**：战斗、走位、技能释放全部自动

### 可选扩展

- **离线收益**：挂机时长换算经验/金币（如 8 小时离线 = 2 小时在线收益）
- **放置收集**：打怪掉落、自动拾取、简单背包

---

## 三、可砍掉的系统（优先级）

### 高优先级砍（大幅减负）

| 系统 | 砍掉后 | 保留最小 |
|------|--------|----------|
| 装备系统 | 装备、强化、镶嵌、品质 | 无，或仅武器/护甲 2 件 |
| 技能树 | 全部 | 无，或被动加成（如攻击+5%） |
| 职业系统 | 全部 | 单职业 |
| 副本/BOSS | 全部 | 无 |
| 公会/组队 | 全部 | 无 |
| 交易/拍卖行 | 全部 | 单机无交易 |
| 多地图 | 全部 | 1–2 张地图即可 |

### 中优先级砍（适度减负）

| 系统 | 砍掉后 | 保留最小 |
|------|--------|----------|
| 完全可破坏 | 红石、自动化、建造 | 树/石头可破坏、采集 |
| 季节/天气 | 全部 | 无，或仅昼夜 |
| 宠物系统 | 全部 | 无 |
| 村庄系统 | 全部 | 无 |

### 保留（已有基础）

- 地形、怪物、角色、可破坏块
- 织梦岛风格视觉
- 传奇地图数据（.map、RespawnInfo）

---

## 四、简化后的 MVP 范围

### 最小可玩版本（2–4 周）

1. **单地图**：传奇 0 地图或 1 张
2. **挂机战斗**：角色自动寻怪、自动攻击、自动拾取
3. **简单属性**：等级、经验、攻击、防御、生命
4. **无装备**：或仅 1 把武器 + 1 件护甲

### 扩展版本（可选）

- 离线收益
- 2–3 张地图切换
- 简单商店（金币买药水/装备）

---

## 五、技术实现简化

- **战斗**：无技能释放、无连击，改为「进入范围 → 自动普攻」
- **AI**：怪物可简单（站桩或原地巡逻），玩家 AI 自动寻怪
- **存档**：仅角色属性 + 地图破坏状态，无装备/技能树
- **网络**：单机优先，多人可延后或砍掉

---

## 六、建议下一步

1. **确认方向**：是否接受「偏挂机、单机、轻数值」的简化？
2. **调整 roadmap**：将「可破坏交互」改为「挂机自动战斗」优先
3. **砍掉 Phase 2**：装备、技能树、经济可延后或砍掉

---

## 七、参考挂机游戏

- **放置类**：Idle Champions、Melvor Idle
- **挂机 RPG**：一念逍遥、剑与远征（简化版）
- **核心**：低操作、自动战斗、数值成长、简单收集
