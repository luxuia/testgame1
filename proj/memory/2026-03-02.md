## 传奇地形项目 - 场景管理流程与性能优化

### 完成内容

#### 1. 场景管理流程（阶段一）
- **BlockRegistry**：方块坐标索引，Register/Unregister/GetBlockAt
- **BlockController**：可破坏块（Wall、Tree、House、Bridge），TakeDamage、OnDestroy 自动 Unregister
- **TerrainGenerator**：运行时从 .map 生成地形，支持 Generate 全图与 GenerateChunk 分块
- **LegendarySceneManager**：场景入口，协调地形、怪物、角色，子节点 Terrain/Monsters/Character
- **MonsterSpawner + MonsterController**：按 RespawnInfo 实例化怪物，MonsterPrefabConfig 映射
- **LegendaryCharacterController**：玩家 WASD 移动
- **Character_Player Prefab**：Capsule 占位，Create Block Prefabs 生成

#### 2. 大场景性能优化（阶段二）
- **TerrainChunk + TerrainChunkManager**：16×16 分块，LoadRadius=2、UnloadRadius=3，按玩家距离加载/卸载
- **ChunkMeshBuilder**：不可破坏块按材质合并，单块 1–2 Draw Call
- **GPU Instancing**：材质 enableInstancing = true
- **DistanceCulling**：怪物超出 30m 禁用 Renderer/Collider
- **分帧加载**：_pendingLoads 队列，LateUpdate 每帧加载一块

#### 3. 视觉升级（织梦岛风格）
- 贴图：32×32 程序化（草地、石砖、木纹、木板、半透明水）
- 模型：树=圆柱+球体，房子=立方+四棱锥屋顶
- CreateLegendaryBlockPrefabs：贴图→材质→Prefab，每次覆盖确保绑定

#### 4. 地图解析扩展（此前完成）
- Mir2Cell：Back/Middle/Front 贴图索引、FishingCell
- Mir2MapParser：v0–v7、v100 全格式
- TileIndexMapper：索引→Tree/House/Bridge
